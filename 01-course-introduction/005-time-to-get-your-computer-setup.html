This course was filmed using the software pgAdmin 3, then later on pgAdmin 4 came out in beta. However it was buggy for a lot of students, so we decided to wait until it was more stable. There are still some bugs, but it is stable enough that we will show you how to install pgAdmin 4 with PostgreSQL 9.6 in the next lecture so you can use it for the course. 

Fortunately the installation of pgAdmin 4 and PostgreSQL is essentially the same for both Mac OS and Windows Users, so you should be able to follow along with the next lecture, just make sure to set aside about 20-30 minutes to complete the set-up process. If you are using a Mac OS computer however, you may find pgAdmin 4 to be very buggy, in which case I recommend installing pgAdmin 3 and PostgreSQL 9.5, check out the special notes below.

Once you are done with this installation you can skip to Section 4. While pgAdmin 3 and 4 look different, all the functionality is essentially the same, and all your SQL code will be the same as well. The main difference is that in pgAdmin 4 the query button is a lightening rod (shown in Lecture 6) and in pgAdmin 3 the button looks like a magnifying glass.

Special Notes for Mac OS users and pgAdmin 4 bugs: 

* pgAdmin 4 can be extremely buggy for Mac Users, while most Mac Users can get it working after a lot of workarounds, pgAdmin 3 is much more stable for Mac Users. I would recommend using pgAdmin 3 if you are a Mac User, you can find it here: https://www.pgadmin.org/download/macos.php . Even though this site says pgAdmin 3 is no longer supported, it still works just fine. You can skip to the Mac Installation section of the curriculum to watch me go through pgAdmin 3 installation. 

If you do want to continue with pgAdmin 4 on a Mac, here are some trouble-shooting tips:

* Some Mac users will encounter a bug not allowing them to find the dvdrental.tar file. If you encounter this issue, check out the advice here: http://stackoverflow.com/questions/39796268/errno-2-no-such-file-or-directory-when-trying-to-backup-a-database-in-pgadmi

* One of the main points of advice on that page says to download an updated version of pgAdmin 4, which you can find here: https://www.pgadmin.org/download/macos4.php . I've also seen that actually downgrading to pgAdmin 4.0 fixes some issues.

* For some Mac users, when restoring the database, it may look to take a really long time (Longer than 30 seconds), if this happens to you, try restarting pgAdmin 4 and then clicking refresh the database, it should then be up and running.

* For some Mac users, the file path to the bin folder (mentioned in the install lecture) is under a hidden folder, try checking these locations for it (you'll need to display hidden folders to see this file as the usr folder is hidden):

Could it be under:

/usr/local/postgres 

or

/usr/local/var/postgres 

or

/Library/PostgreSQL/9.5 

Got questions? Check out the FAQ, if its not answered there then post to the QA forums. You may also find it helpful to search the previous QA posts on this topic.
